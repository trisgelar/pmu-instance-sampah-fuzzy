Project Overview

What is PMU Instance Sampah Fuzzy?

PMU Instance Sampah Fuzzy is a comprehensive machine learning project focused on trash image segmentation using YOLO (You Only Look Once) models. The project supports multiple YOLO versions (v8, v10, v11) and provides automated pipeline scripts for training, inference, and model export.

Key Features

  • Multi-YOLO support (v8, v10, v11)
  • Automated pipeline scripts
  • ONNX and RKNN model export
  • Comprehensive testing framework
  • Google Drive integration
  • Virtual environment management
  • Colored output and progress tracking

Project Structure

pmu-instance-sampah-fuzzy/ ├── run_yolov8_pipeline.sh # YOLOv8 pipeline ├── run_yolov10_pipeline.sh # YOLOv10 pipeline ├── run_yolov11_pipeline.sh # YOLOv11 pipeline ├── run_all_pipelines.sh # Master pipeline ├── main_colab.py # Main orchestration ├── modules/ # Core modules ├── tests/ # Testing framework ├── docs/ # Documentation ├── config.yaml # Configuration └── compressed/ # Output files

Installation Guide

Prerequisites

  • Python 3.11+
  • CUDA-compatible GPU (optional)
  • Git
  • Node.js (for TiddlyWiki-style docs)

Setup Steps

1. Clone the Repository

git clone https://github.com/your-repo/pmu-instance-sampah-fuzzy.git cd pmu-instance-sampah-fuzzy

2. Create Virtual Environment

python -m venv .venv source .venv/bin/activate # On Windows: .venv\Scripts\activate

3. Install Dependencies

pip install -r requirements.txt

4. Make Scripts Executable

chmod +x run_*.sh
Note: The project includes automated scripts that handle virtual environment activation and dependency management.
Home > Quick Start

Quick Start Guide

1. Check Available Options

./run_all_pipelines.sh help

2. Use Existing Results (Recommended)

./run_yolov8_pipeline.sh use_existing

3. Train New Model

./run_yolov8_pipeline.sh train_new

4. Run Complete Pipeline

./run_yolov8_pipeline.sh complete_pipeline
Success! You should now see training results, analysis, and model exports in the respective directories.

Home > Pipeline Scripts > YOLOv8 Pipeline

YOLOv8 Pipeline

Usage

./run_yolov8_pipeline.sh [scenario]

Available Scenarios

  • train_new - Train new YOLOv8 model
  • use_existing - Use existing results
  • complete_pipeline - Full pipeline
  • safe_pipeline - Safe execution
  • onnx_export - Export to ONNX
  • analysis_only - Analysis only
  • inference_only - Inference only
  • rknn_only - RKNN conversion
  • all_models - All variants
  • custom - Custom configuration

Configuration

MODEL_VERSION="v8n" # Change to v8s, v8m, v8l, v8x EPOCHS=50 BATCH_SIZE=16 IMAGE_SIZE=640 DATASET_PATH="datasets" SAVE_TO_DRIVE=true

Common Issues & Solutions

Permission Denied

Error: Permission denied when running scripts
chmod +x run_*.sh

Virtual Environment Not Found

Error: Virtual environment not detected
source .venv/bin/activate

Missing Dependencies

Error: ModuleNotFoundError: No module named 'yaml'
pip install -r requirements.txt
