Project Overview

What is PMU Instance Sampah Fuzzy?

PMU Instance Sampah Fuzzy is a comprehensive machine learning project focused on trash image segmentation using YOLO (You Only Look Once) models. The project supports multiple YOLO versions (v8, v10, v11) and provides automated pipeline scripts for training, inference, and model export.

Key Features

  • Multi-YOLO support (v8, v10, v11)
  • Automated pipeline scripts
  • ONNX and RKNN model export
  • Comprehensive testing framework
  • Google Drive integration
  • Virtual environment management
  • Colored output and progress tracking

Project Structure

Installation Guide

Prerequisites

  • Python 3.11+
  • CUDA-compatible GPU (optional)
  • Git

Setup Steps

1. Clone the Repository

git clone https://github.com/your-repo/pmu-instance-sampah-fuzzy.git cd pmu-instance-sampah-fuzzy

2. Create Virtual Environment

python -m venv .venv source .venv/bin/activate # On Windows: .venv\Scripts\activate

3. Install Dependencies

pip install -r requirements.txt

4. Make Scripts Executable

chmod +x run_*.sh
Note: The project includes automated scripts that handle virtual environment activation and dependency management.

Quick Start Guide

1. Check Available Options

./run_all_pipelines.sh help

2. Use Existing Results (Recommended)

./run_yolov8_pipeline.sh use_existing

3. Train New Model

./run_yolov8_pipeline.sh train_new

4. Run Complete Pipeline

./run_yolov8_pipeline.sh complete_pipeline
Success! You should now see training results, analysis, and model exports in the respective directories.

Pipeline Scripts Overview

Available Scripts

  • run_yolov8_pipeline.sh - YOLOv8 specific pipeline
  • run_yolov10_pipeline.sh - YOLOv10 specific pipeline
  • run_yolov11_pipeline.sh - YOLOv11 specific pipeline
  • run_all_pipelines.sh - Master script for all versions

Common Scenarios

Scenario Description Command
Train New Train from scratch train_new
Use Existing Use trained models use_existing
Complete Pipeline Full workflow complete_pipeline
Safe Pipeline Skip problematic steps safe_pipeline

YOLOv8 Pipeline

Usage

./run_yolov8_pipeline.sh [scenario]

Available Scenarios

  • train_new - Train new YOLOv8 model
  • use_existing - Use existing results
  • complete_pipeline - Full pipeline
  • safe_pipeline - Safe execution
  • onnx_export - Export to ONNX
  • analysis_only - Analysis only
  • inference_only - Inference only
  • rknn_only - RKNN conversion
  • all_models - All variants
  • custom - Custom configuration

Configuration

MODEL_VERSION="v8n" # Change to v8s, v8m, v8l, v8x EPOCHS=50 BATCH_SIZE=16 IMAGE_SIZE=640 DATASET_PATH="datasets" SAVE_TO_DRIVE=true

Master Pipeline

Usage

./run_all_pipelines.sh [version] [scenario]

Unified interface for all YOLO versions. Examples:

./run_all_pipelines.sh v8 train_new ./run_all_pipelines.sh v10 use_existing ./run_all_pipelines.sh all complete_pipeline

Training Scenarios

train_new

Train a new model from scratch with the following steps:

  • Download dataset from Roboflow
  • Initialize YOLO model
  • Train for specified epochs
  • Save training results

Inference Scenarios

inference_only

Run inference and visualization on existing models:

  • Load trained model
  • Run inference on test images
  • Generate visualizations
  • Save results

Export Scenarios

onnx_export

Export PyTorch model to ONNX format for deployment.

rknn_only

Convert model to RKNN format for RKNN-compatible devices.

Custom Scenarios

custom

Use custom configuration with modified parameters. Edit the script to change:

CUSTOM_EPOCHS=100 CUSTOM_BATCH_SIZE=8 CUSTOM_IMAGE_SIZE=512

Common Issues & Solutions

Permission Denied

Error: Permission denied when running scripts
chmod +x run_*.sh

Virtual Environment Not Found

Error: Virtual environment not detected
source .venv/bin/activate

Missing Dependencies

Error: ModuleNotFoundError: No module named 'yaml'
pip install -r requirements.txt

ONNX/RKNN Issues

ONNX Export Issues

Common issues with ONNX export and their solutions:

  • Verify model compatibility

RKNN Conversion Issues

Common issues with RKNN conversion and their solutions:

  • Check CUDA compatibility
  • Verify target platform
